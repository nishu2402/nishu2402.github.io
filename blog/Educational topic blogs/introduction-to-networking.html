Network Basics & Types

A network is two or more devices connected together to share resources and communicate using a defined set of rules (protocols). The internet is the ultimate example — a "network of networks" where millions of private networks interconnect through standardised protocols to create global connectivity. Every device on a network has a unique identifier (MAC address at Layer 2, IP address at Layer 3) that enables data to reach its intended destination.

The OSI Model — 7 Layers

The OSI (Open Systems Interconnection) model is the universal reference framework for how network communication works, divided into seven distinct layers. Each layer has a specific function and communicates only with the layers directly above and below it. The OSI model is the language every network engineer, security analyst, and penetration tester uses to describe, troubleshoot, and categorise network behaviour. Understanding which layer a vulnerability exists at immediately tells you the appropriate mitigation.

Mnemonic — All People Seem To Need Data Processing (Application → Physical, top-down) or Please Do Not Throw Sausage Pizza Away (Physical → Application, bottom-up).

Network Devices & Media

📡
Hub (Layer 1)
Broadcasts all traffic to every connected port — a "dumb" repeater. Every device sees every packet regardless of intended recipient. Creates a collision domain. Largely obsolete — replaced by switches. Still useful for passive traffic capture in lab environments.
🔄
Switch (Layer 2)
Learns MAC addresses on each port and builds a CAM table — forwards frames only to the correct destination port. Reduces collisions, improves performance. Managed switches support VLANs, port mirroring (SPAN), 802.1X NAC, and QoS. Unmanaged switches require zero configuration.
🌐
Router (Layer 3)
Routes packets between different networks using IP addresses and a routing table. Connects LAN to WAN, performs NAT (hiding internal IPs behind one public IP), provides basic firewall/ACL filtering, and handles dynamic routing protocols (OSPF, BGP). The boundary between networks.

Network Topologies

Network topology describes the physical or logical arrangement of nodes and connections. Physical topology is how cables and devices are laid out in the real world. Logical topology is how data flows through the network — which can differ from the physical layout. For example, a physical star topology can operate as a logical bus. Understanding topology is essential for both network design and for mapping attack paths during security assessments.

Bus
All nodes on one cable. Simple but a single break disrupts the entire network. Legacy Ethernet (10BASE2).
Star
All nodes connect to a central switch. Most common modern topology. One node failure doesn't affect others.
Ring
Data travels in one direction around the ring. Token Ring is the classic example. A break disrupts the whole loop.
Mesh
Every node connects to every other node. Maximum redundancy. Used in internet backbone and critical infrastructure.

IP Addressing, CIDR & Subnetting

Every device on a network needs a unique address — just like a postal address — to send and receive data. IPv4 addresses are 32-bit numbers written in dotted-decimal notation (e.g., 192.168.1.10). An IPv4 address has two parts: the Network ID (identifying which network) and the Host ID (identifying the device within that network). The subnet mask determines the boundary between these two parts.

IPv6 — 128-Bit Addressing

IPv4 ran out of addresses in 2011. IPv6 uses 128-bit addresses written as eight groups of four hexadecimal digits separated by colons: 2001:0db8:85a3:0000:0000:8a2e:0370:7334. Consecutive groups of all zeros can be compressed to :: once per address. IPv6 provides 340 undecillion (3.4 × 10³⁸) addresses — enough for every atom on Earth to have a unique address.

Core Protocols & the TCP Handshake

Protocols are the rules governing how data is formatted, transmitted, and received between network devices. Understanding protocols at a packet level — not just their names — is what separates a network administrator from a network security engineer. Knowing exactly how TCP establishes connections, how DNS resolves names, and how DHCP assigns addresses is prerequisite knowledge for understanding how each of these protocols can be attacked.

TCP 3-Way Handshake — Connection Establishment
CLIENT
SERVER
① SYN — seq=0 — "I want to connect"
② SYN-ACK — seq=0, ack=1 — "Acknowledged, I'm listening"

[ i ]

Security Note: SYN flood attacks send millions of SYN packets without completing the handshake, exhausting the server's connection table. Nmap's SYN scan (-sS) sends SYN, receives SYN-ACK (confirming port open), then sends RST instead of ACK — never completing the handshake and leaving less forensic evidence.

Wireless Networking, VLANs & VPNs

Three modern networking fundamentals that were absent from legacy curricula but are now essential: wireless networking (Wi-Fi) which is how most devices connect today, VLANs which logically segment physical networks for security and performance, and VPNs which securely extend private networks over public infrastructure. Every security practitioner needs to understand all three — both how they work correctly and how they are commonly misconfigured.

VPN Types
A Virtual Private Network encrypts traffic between two points over an untrusted public network. Site-to-Site VPN: connects two entire networks (HQ ↔ Branch office). Remote Access VPN: connects an individual device to a corporate network (employee laptop → corporate LAN). Protocols: OpenVPN, IPSec/IKEv2, WireGuard (modern, fast), SSTP, L2TP. WireGuard is now the recommended standard for new deployments.
NAT — Network Address Translation
NAT allows an entire private network to share one public IP address. The router maintains a translation table: when Device 192.168.1.10 sends a packet to the internet, the router rewrites the source IP to its public IP and records the mapping. When the response arrives, it rewrites the destination back to 192.168.1.10. SNAT (Source NAT) for outbound. DNAT (Destination NAT / Port Forwarding) for inbound — routes external traffic to a specific internal host.
Dynamic Routing Protocols
Routers use dynamic routing protocols to automatically share network maps. OSPF (Open Shortest Path First) is the standard interior gateway protocol — used within an organisation. BGP (Border Gateway Protocol) is the internet's routing protocol — used between autonomous systems (ISPs, large organisations). BGP route injection is a known attack vector — a misconfigured BGP router can redirect global internet traffic.

Network Security Fundamentals

Network security is built on layered defences — no single control protects against all threats. Understanding firewalls, packet filtering, NAC, and basic troubleshooting tools provides both the defensive foundation and the prerequisite knowledge for understanding how each control can be bypassed during a penetration test.

ipconfig / ifconfig
Windows / Linux
Display IP configuration, subnet mask, default gateway, DNS. Pen test: confirm network position and reachable subnets
ping
ICMP · Connectivity
Test host reachability via ICMP echo request/reply. Pen test: host discovery in subnets. Can be blocked by firewall
tracert / traceroute
Path Tracing
Show each hop between source and destination with latency. Pen test: map network topology from a foothold
nmap
Port Scanner
Discover hosts, open ports, services, OS. The primary network reconnaissance tool for every penetration test
arp -a / ip neigh
ARP Table
Show IP-to-MAC address mappings. Pen test: discover local network hosts without active scanning
netstat / ss
Connections
Show active connections, listening ports, routing table. Essential for post-exploitation situational awareness
Wireshark
Packet Capture
Capture and analyse packets. Decode TCP handshakes, read cleartext protocols, identify network anomalies
dig / nslookup
DNS
Query DNS records. Pen test: DNS enumeration, zone transfer (AXFR), subdomain discovery

AI Integration in Networking

Artificial Intelligence is transforming networking across every domain — from how networks are designed and operated to how they are attacked and defended. Traditional networking relied on manual configuration, static rules, and human analysis of logs. AI introduces dynamic adaptation: networks that reconfigure themselves in response to traffic patterns, security systems that detect anomalies no rule could describe, and automated operations that manage network complexity at a scale no human team could maintain.

For security practitioners, AI networking tools are now essential on both sides of every engagement — AI-assisted network discovery and attack path analysis on the offensive side, and AI-powered anomaly detection and automated response on the defensive side.

[ AI ]

AI-powered network anomaly detection systems can identify novel attack patterns — new malware C2 protocols, lateral movement sequences, data exfiltration behaviours — that have never been seen before and have no signature, purely by detecting statistical deviation from the learned baseline of normal network behaviour. This capability is now the primary detection layer in enterprise SOCs against advanced persistent threats.

🧠
AI Network Anomaly Detection
ML models baseline every device's normal communication patterns — connection destinations, protocols used, data volumes, timing — and flag deviations in real time. Detects novel attacks with no known signature: new C2 protocols, unusual east-west lateral movement, DNS tunnelling, and data exfiltration patterns that rule-based IDS systems completely miss.
🗺
AI Network Topology Mapping
AI tools like Darktrace and Armis passively learn the complete network topology — every device, every connection, every protocol — without requiring network diagrams or manual inventory. Produces a live, continuously-updated network map that reflects the actual state of the environment, not what the asset register claims should be there.
⚡
AI-Driven Network Operations (AIOps)
AI network management platforms predict and prevent network failures before they occur — analysing traffic patterns to predict congestion, automatically optimising routing to maintain QoS, detecting hardware degradation from performance metrics, and provisioning capacity in advance. Reduces mean-time-to-detect and mean-time-to-resolve network incidents by orders of magnitude.
🔍
Automated Pen Test Network Recon
AI tools correlate Nmap scan output, ARP tables, routing data, and OSINT to automatically build a comprehensive internal network model — identifying subnets, device roles, trust relationships, and attack paths. Compresses hours of manual network mapping into minutes, and identifies non-obvious lateral movement paths that human analysis misses in complex multi-segment environments.
🛡
AI-Powered NDR
Network Detection and Response (NDR) platforms (Darktrace, Vectra AI, ExtraHop) use unsupervised ML to learn what is normal for every device and alert on deviations. Unlike SIEM rules, they require no prior knowledge of attack techniques — detecting completely novel attack methods from first principles by recognising behaviour that deviates from the learned baseline.
☁️
AI Cloud Network Security
AWS GuardDuty, Azure Defender for Cloud, and GCP Security Command Center use ML to analyse VPC flow logs, DNS query logs, and API calls for threat indicators — detecting compromised EC2 instances, unusual data exfiltration, cryptocurrency mining, and lateral movement through cloud IAM permissions that traditional network monitoring cannot detect.
🤖
AI Network Configuration Management
AI tools continuously audit network device configurations against security baselines — detecting drift from expected configurations, identifying new routes that shouldn't exist, flagging firewall rule changes that create unintended access, and automatically remediating misconfiguration. Prevents the slow erosion of network security posture that manual change management cannot keep pace with at enterprise scale.
📡
AI for Wireless Security
ML models analyse Wi-Fi traffic patterns to detect rogue access points (same SSID, different BSSID), evil-twin attacks, deauthentication flood attacks, and unusual client association patterns. AI systems can distinguish between legitimate AP signal variation and active wireless jamming or spoofing attacks that passive monitoring systems cannot identify.
🔮
Predictive Network Security
AI correlates vulnerability data, threat intelligence, and network topology to predict which network paths are most likely to be exploited in the next attack — enabling proactive firewall rule tightening, segmentation improvements, and patch prioritisation before an attacker discovers the same paths through reconnaissance. Converts reactive security operations into proactive risk management.
