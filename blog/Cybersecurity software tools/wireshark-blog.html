Installation & Interface Setup

Wireshark is the world's most widely used open-source network protocol analyser, developed by Gerald Combs and maintained by a global community. It captures live network packets from any interface, dissects them into their component protocol fields, and presents them in a colour-coded, filterable GUI that makes complex traffic analysis accessible to security professionals at every level. Wireshark is pre-installed in Kali Linux and is freely available for Windows, macOS, and Linux from wireshark.org.

Wireshark operates in two modes: live capture directly from a network interface (requires elevated privileges), and PCAP file analysis (opening saved packet capture files -- no privileges required). The companion tool TShark provides full Wireshark capabilities from the command line, enabling automation, scripting, and headless analysis in server environments.

01
Download and Install
Download from wireshark.org/download.html. On Windows: run the installer -- it includes Npcap (the packet capture driver) automatically. On Kali/Ubuntu: sudo apt install wireshark -- when prompted whether non-superusers should capture, select Yes to allow your user to capture without root. On macOS: brew install --cask wireshark.
Linux: add your user to the wireshark group: sudo usermod -aG wireshark $USER (then log out and back in)
02
The Wireshark Main Window
When Wireshark opens you see three areas: the Packet List pane (top -- every captured packet as a row), the Packet Details pane (middle -- expandable protocol tree for the selected packet), and the Packet Bytes pane (bottom -- raw hex and ASCII). Above all three is the display filter bar where you type filter expressions to narrow what is shown.
03
Colour Coding
Wireshark colour-codes packets by protocol and status. Default colours: green = TCP, blue = UDP/DNS, light orange = HTTP, dark red = errors/RST packets, yellow = ARP/ICMP. Customise via View → Coloring Rules. Colour rules make at-a-glance traffic pattern recognition dramatically faster.
04
Configure Preferences for Security Analysis
Go to Edit → Preferences. Key settings: Appearance → Layout -- choose a 3-pane layout that suits your workflow. Protocols → TCP -- enable "Allow subdissector to reassemble TCP streams" for proper HTTP/TLS reassembly. Protocols → HTTP -- enable "Reassemble HTTP headers spanning multiple TCP segments". These settings ensure complete protocol analysis of multi-packet conversations.
For TLS decryption: Preferences → Protocols → TLS → set RSA Keys List or (Pre-)Master-Secret log filename if you have the keys.
02 //

Capturing Live Traffic

Live packet capture is the starting point for network analysis, incident response, and penetration testing validation. Wireshark gives you complete control over which interface to capture from, what traffic to collect with capture filters (applied before packets reach Wireshark), and how to manage capture size and rotation for long-running sessions. Understanding the difference between capture filters (applied at capture time -- what gets saved) and display filters (applied after capture -- what you see) is the most important Wireshark concept to master.

🏠 Selecting an Interface
The Wireshark welcome screen shows all available interfaces with a live sparkline of traffic activity. Double-click any interface to start capturing immediately. Key interfaces: eth0/Ethernet (wired), wlan0/Wi-Fi (wireless), lo/Loopback (local machine traffic). On the welcome screen, click the gear icon next to any interface to set capture filters before starting.
📷 Capture Filters (BPF Syntax)
Set under Capture → Options → Capture filter. These use Berkeley Packet Filter syntax and apply before packets are saved -- reducing file size and CPU load. Examples: host 192.168.1.10 (one host only), port 80 or port 443 (web only), not arp (exclude ARP noise), tcp and port 22 (SSH only).
💾 Saving Capture Files
Go to File → Save As to save the current capture. Standard format is .pcapng (preferred, supports metadata) or .pcap (legacy, maximum compatibility). For long-running captures, use Capture → Options → Output to enable ring-buffer rotation -- automatically creates new files every N megabytes or N minutes, retaining only the last K files.
🌐 Remote Capture
Wireshark can capture from a remote interface over SSH. Navigate to Capture → Options → Manage Interfaces → Remote Interfaces. Alternatively, use SSH piping: ssh user@remote "tcpdump -w -" | wireshark -k -i - on Linux/macOS captures remotely and pipes directly into your local Wireshark GUI in real time.
🚩 Stopping and Restarting
Click the red Stop button (square) in the toolbar to halt capture. Click the blue shark-fin Restart button to clear and start fresh. Use Capture → Start (Ctrl+E) to begin and the same shortcut to stop. The status bar at the bottom shows packet count, elapsed time, and capture rate in packets/second.
⛔ Promiscuous Mode
Enabled by default in Capture → Options. In promiscuous mode, the network adapter captures all packets on the segment, not just those addressed to your MAC. Essential for traffic analysis on shared networks. Note: on switched networks, promiscuous mode only captures broadcast/multicast and traffic to/from your host -- you need port mirroring or a hub to capture all segment traffic.

Display Filters -- The Most Powerful Skill

Wireshark display filters are the most critical skill in the entire tool. A raw capture on a busy network can contain millions of packets -- display filters let you isolate exactly the traffic you care about with surgical precision. Wireshark's display filter language is vastly more expressive than the BPF syntax used for capture filters. You can filter on any protocol field, any value, any comparison, any boolean combination, and any string match across the entire protocol dissection tree.

ip.addr == 192.168.1.10
Show all traffic to or from a specific IP address (source OR destination). Use ip.src or ip.dst to restrict to one direction only.
tcp.port == 443 && ip.addr == 10.0.0.5
Combine multiple conditions with && (AND), || (OR), ! (NOT). This shows HTTPS traffic to/from one specific host -- isolating all TLS sessions for that IP.
http.request.method == "POST"
Filter on specific protocol field values. Shows only HTTP POST requests -- invaluable for spotting form submissions, API calls, login attempts, and data exfiltration over plain HTTP. Combine with http contains "password" to find credential submissions.
dns.qry.name contains "malicious"
String matching on protocol fields. Filter DNS queries containing specific strings -- use to spot C2 domain generation algorithm (DGA) traffic, suspicious TLD lookups, or data exfiltration via DNS tunnelling. Try dns.qry.name matches ".*\.tk$" for regex matching against suspicious TLDs.
tcp.flags.syn == 1 && tcp.flags.ack == 0
Show only TCP SYN packets (connection initiation -- no ACK). This isolates all new connection attempts from a host. Use to detect port scanning (many SYN packets to different ports), SYN flood attacks, or map all outbound connections from a suspicious host.
!(arp || dns || icmp)
Exclude common noisy protocols to focus on application traffic. The ! operator negates and parentheses group. This removes ARP, DNS, and ICMP -- dramatically reducing noise in a busy capture to expose actual application-layer conversations.
frame.time_delta > 1
Filter on timing. Shows packets where the inter-packet gap exceeds 1 second -- useful for identifying connection timeouts, network slowdowns, slow data exfiltration (beaconing), or suspicious pause-and-send patterns characteristic of malware C2 traffic.
tcp.analysis.flags && !tcp.analysis.window_update
Show TCP analysis events (retransmissions, duplicate ACKs, out-of-order) while excluding window updates. Wireshark's TCP analysis engine automatically detects network problems -- this filter surfaces them. Essential for diagnosing slow networks, packet loss, and connection quality issues.
[ * ]

Pro Tip -- Bookmark Your Filters: Right-click the display filter bar and select "Save this filter" to add frequently used filters to your bookmarks (the bookmark icon left of the filter bar). Build a library of security-relevant filters: C2 detection, port scan identification, credential capture patterns, and protocol anomalies. Saved filters are available in one click for every future session.

Security analysis focuses on a core set of protocols across the OSI stack -- from ARP at Layer 2 through to application-layer protocols like HTTP, DNS, SMB, and FTP. Understanding what each protocol's packets look like in the normal case is what allows you to spot the anomalies that indicate attacks, malware, misconfigurations, and data exfiltration.

Stream Reconstruction & Statistics

Individual packets tell you what was transmitted; stream reconstruction tells you the full conversation. Following a TCP or UDP stream reassembles all related packets into the complete data exchange -- showing you an entire HTTP request and response, a complete FTP file transfer, or the full content of a telnet session in human-readable form. Combined with Wireshark's statistics tools, stream reconstruction is the fastest way to extract meaningful intelligence from a capture.

01
Follow TCP Stream
Right-click any packet in a TCP conversation → Follow → TCP Stream. A new window shows the entire conversation with client data in one colour and server data in another. You can see complete HTTP requests and responses, FTP commands and data, Telnet sessions, and any other plaintext TCP protocol in full. Use the Show and save data as dropdown to change encoding (ASCII, hex, raw, C arrays).
Every TCP stream gets a stream index number. The display filter bar automatically updates to show only that stream's packets: tcp.stream eq N
02
Follow UDP / HTTP / TLS Streams
Same right-click menu provides: Follow UDP Stream (DNS queries and responses, VoIP, custom UDP protocols), Follow HTTP Stream (reconstructs HTTP request + response pair), and Follow TLS Stream (shows encrypted data -- decrypts if you have provided keys in Preferences). Each stream type reassembles its specific protocol's data correctly.
03
Export Objects -- Recover Files
Navigate to File → Export Objects. Choose protocol: HTTP (extract files downloaded over HTTP -- images, scripts, executables, documents), SMB (files transferred over the network), FTP-DATA (files transferred via FTP). Wireshark reassembles multi-packet file transfers and presents each file for export. Critical for malware sample extraction in incident response.
Recovered files can be submitted to VirusTotal or analysed with a sandbox directly from Export Objects.
04
Conversations and Endpoints Statistics
Navigate to Statistics → Conversations. The Conversations window shows every unique two-host communication pair with bytes transferred, packet counts, and duration. Sort by bytes to quickly find the largest data transfers -- potential exfiltration. The Endpoints view (Statistics → Endpoints) shows all unique hosts seen in the capture, sortable by traffic volume.
05
I/O Graph and Protocol Hierarchy
Statistics → I/O Graph plots packet rate or bytes/second over time -- spots traffic bursts, attack timing, and beaconing intervals as visual patterns. Statistics → Protocol Hierarchy shows a percentage breakdown of all protocols in the capture -- immediately reveals unexpected protocols (e.g. Telnet in a network that should only have SSH, or IRC which often indicates malware C2).

Network Forensics & Attack Detection

Wireshark is a primary network forensics tool -- PCAP files are evidence. Analysing network captures during and after security incidents reveals attacker behaviour, data exfiltration paths, lateral movement, and malware C2 communication patterns that endpoint-only forensics cannot provide. The combination of Wireshark's protocol dissection, stream reconstruction, and statistical analysis produces network-level intelligence that is critical for complete incident reconstruction.

🔎 Detecting ARP Spoofing
Filter: arp.duplicate-address-detected. Wireshark automatically flags when two different MAC addresses claim the same IP. In the Packet Details, look for "Duplicate IP address detected" warnings. ARP spoofing attacks generate a stream of gratuitous ARP replies trying to redirect traffic through the attacker's machine for MITM interception.
📈 Port Scan Detection
Use Statistics → Conversations sorted by packet count to spot scanners -- many short TCP conversations to different destination ports from one source. Then filter: tcp.flags.syn==1 && ip.src==ATTACKER_IP. The pattern of SYN followed by RST/ACK (closed port) or SYN-ACK (open port) identifies the scan type (SYN, Connect, FIN etc.).
👻 Malware C2 Traffic
Look for: regular periodic outbound connections (beaconing) visible in Statistics → I/O Graph as repeating spikes. Filter DNS for DGA-style names: dns.qry.name matches "^[a-z0-9]{12,}". Unusual protocols on non-standard ports (IRC on port 6667, HTTP on port 4444). Large DNS TXT record responses indicating DNS tunnelling.
📄 Credential Capture
For plaintext protocols, Follow Stream immediately reveals credentials. Filter: http.authorization for HTTP Basic Auth, ftp.request.command == "PASS" for FTP, smtp.req.parameter contains "AUTH" for email. Follow the TCP stream to see the full authentication exchange including the username and password in cleartext.
⚡ Data Exfiltration
In Statistics → Conversations, sort by bytes transferred. Unusually large outbound transfers to external IPs deserve investigation. Filter by the top destination: ip.dst == SUSPECT_IP and follow the stream to see what data was transferred. File → Export Objects reconstructs transferred files from HTTP/SMB/FTP sessions.
🔒 TLS Certificate Analysis
Even without decrypting TLS, filter: tls.handshake.type == 11 (Certificate message). In the Packet Details, expand TLS → Handshake Protocol → Certificate to see the server certificate -- subject, issuer, validity period, and Subject Alternative Names. Malware C2 often uses self-signed certificates with suspicious or generic subjects that stand out from legitimate HTTPS traffic.

TShark -- Command Line Power

TShark is the terminal-mode companion to Wireshark, sharing all the same dissectors, filters, and protocol support but operating without a GUI. TShark is essential for security automation -- capturing on headless servers, processing PCAP files in scripts, integrating with SIEM pipelines, and feeding packet data into AI analysis tools. Every display filter that works in Wireshark works identically in TShark with the -Y flag.

# Live capture on interface eth0, write to file tshark -i eth0 -w capture.pcapng # Read PCAP and apply display filter tshark -r capture.pcap -Y "http.request.method == POST" # Extract specific fields as CSV tshark -r capture.pcap -T fields -e frame.time -e ip.src -e ip.dst -e tcp.dstport -e http.request.uri -Y "http" -E separator=, # Extract all DNS queries from a PCAP tshark -r capture.pcap -Y dns -T fields -e dns.qry.name -e dns.a | sort -u # Reconstruct TCP stream N to file tshark -r capture.pcap -Y "tcp.stream eq 3" -T fields -e data.data | xxd -r -p > stream3.bin # Count packets per source IP (top talkers) tshark -r capture.pcap -T fields -e ip.src | sort | uniq -c | sort -rn | head -20 # Export HTTP objects (files) to directory tshark -r capture.pcap --export-objects http,./extracted_files/ # Capture only specific traffic to PCAP (capture filter) tshark -i eth0 -f "tcp port 80 or port 443" -w web_only.pcap # Output as JSON for pipeline processing tshark -r capture.pcap -T json -Y "dns" > dns_packets.json

AI Integration in Wireshark Analysis

AI transforms Wireshark from a manual inspection tool into an intelligent network threat detection platform. The critical pipeline: TShark extracts structured data from PCAP files at machine speed, outputs JSON or CSV, and feeds that structured traffic data into an LLM that reasons about attack patterns, anomalies, and protocol violations at a level no manual analysis can match at scale. AI identifies the subtle behavioural patterns -- beaconing intervals, DNS anomaly scores, unusual data volumes -- that indicate threats in traffic too large for human review.

[ AI ]

Security analysts using AI-augmented PCAP analysis identify network-based threats 4-6x faster than manual Wireshark review on large captures. AI correlates traffic patterns across thousands of connections simultaneously, identifying C2 beaconing, DNS tunnelling, and lateral movement chains that would take hours to spot manually.

🔎
Traffic Pattern Analysis
AI analyses TShark-extracted conversation statistics, DNS queries, and protocol distribution data to identify attack patterns -- C2 beaconing (regular intervals visible in I/O graph data), DNS tunnelling (high-entropy long domain names), port scan signatures (many short-lived connections from one source), and lateral movement chains (internal host-to-host connection patterns). Processes captures too large for manual review in seconds.
📄
PCAP Report Generation
AI converts TShark-extracted packet metadata into professional incident analysis reports -- identifying the attack timeline, attacker IP behaviour, data exfiltration volumes, and affected systems from raw network captures. Generates findings suitable for technical reports, executive summaries, and legal evidence documentation without the analyst needing to manually trace every conversation.
👻
Malware Traffic Detection
Feed extracted DNS queries, HTTP user agents, TLS certificate fields, and connection patterns to an LLM trained on threat intelligence. AI identifies known malware communication signatures, suspicious DGA domain patterns, unusual protocol usage (IRC on port 443, HTTP on 8888), and C2 callback fingerprints -- correlating against known threat actor TTPs to provide attribution context beyond simple IOC matching.
🛠️
Filter Generation
LLMs generate precise Wireshark display filter expressions from natural-language descriptions. "Show me all outbound connections to non-RFC1918 addresses that are not HTTP or HTTPS" produces the exact Wireshark filter syntax immediately. Eliminates the need to memorise complex field names and filter syntax for infrequently-used protocols -- just describe what you want to see.
📊
Anomaly Detection Pipeline
TShark extracts per-minute connection counts, bytes transferred, unique destinations, and DNS request rates to structured JSON. AI baselines normal behaviour from historical captures and flags statistical anomalies -- a host that normally makes 10 DNS queries/minute suddenly making 500 is an immediate indicator of infection or data exfiltration via DNS tunnelling, caught automatically without manual thresholds.
⚡
Protocol Decode Assistance
When encountering unfamiliar or custom protocols in a capture, AI analyses the raw hex data and packet structure to suggest what the protocol might be, what each field likely represents, and how to write a Wireshark dissector or Lua plugin to properly decode it. Invaluable for malware that uses custom binary C2 protocols or for reverse-engineering undocumented proprietary network applications.
