Installation & Initial Configuration

Tenable Nessus is the world's most widely deployed vulnerability scanner, trusted by over 30,000 organisations globally. It identifies vulnerabilities, misconfigurations, malware, and compliance violations across network devices, operating systems, databases, web applications, and cloud infrastructure. Nessus operates by deploying over 180,000 plugins -- small purpose-built scripts that test for specific vulnerabilities, misconfigurations, and compliance controls -- against target systems during a scan.

Nessus Essentials (free) scans up to 16 IP addresses and is sufficient for learning, home lab use, and small environments. Nessus Professional removes IP limits, adds compliance auditing, live results, and advanced scanning capabilities. Tenable.io and Tenable Security Center are enterprise platforms providing centralised management, continuous monitoring, and team workflows.

01
Download Nessus
Visit tenable.com/products/nessus/select-your-operating-system. Choose your platform: Linux (.deb for Kali/Ubuntu, .rpm for RHEL/CentOS), Windows (.msi), macOS (.dmg), or Docker. Register at tenable.com for a free Nessus Essentials activation code -- covers up to 16 IPs and is fully functional for learning and labs.
02
Install the Service
On Kali/Debian: install the .deb package, then start the service with sudo systemctl start nessusd. On Windows: run the .msi installer -- the service starts automatically. On macOS: open the .dmg and follow the installer. Docker: pull tenableofficial/nessus and run with -p 8834:8834.
Enable auto-start on Linux: sudo systemctl enable nessusd
03
Access the Web Interface
Open your browser and navigate to https://localhost:8834. Accept the self-signed certificate warning (expected and safe for localhost). Create your administrator account, select "Nessus Essentials" or enter a Professional licence key, then enter your activation code.
04
Wait for Plugin Initialisation
After activation, Nessus downloads its entire plugin database -- over 180,000 plugins. This takes 10-30 minutes. A progress bar shows on screen. Do not interrupt. Nessus cannot perform scans until the initial plugin download and compilation completes.
05
Post-Install Configuration
Go to Settings → Software Update and trigger a manual update for the latest plugins. Configure Settings → SMTP Server for scan completion email alerts. Create a dedicated scan operator account under Users -- avoid using the admin account for daily scanning.

Scan Policies & Templates

Scan policies define what Nessus checks for, how aggressively it scans, and what credentials it uses to authenticate. Using the right scan template for the right context is one of the most important Nessus skills -- the wrong template wastes time, misses vulnerabilities, or causes performance issues on production systems. Nessus ships with over 20 pre-built templates covering everything from basic network discovery to deep application testing and compliance auditing.

Basic Network Scan
Most Common Starting Point
General-purpose network vulnerability scan testing ~10,000 plugins across all common services. Supports credentials. Best for initial VA engagements, SMB networks, and external assessments. Navigate: Scans → New Scan → Basic Network Scan.
Advanced Scan
Full Control
Full customisation of every scan parameter -- port ranges, plugin families, performance settings, credentials, and assessment intensity. Use when you need precise control. Navigate: Scans → New Scan → Advanced Scan.
Credentialed Patch Audit
Internal Compliance
Checks for missing OS patches and outdated installed software using administrator credentials. Provides the most accurate patch gap analysis. Best for internal compliance and patch management validation assessments.
Web Application Tests
Web Servers
Targets web servers and applications. Checks HTTP headers, SSL/TLS configuration, CGI flaws, and common web vulnerabilities. Use as a complement to manual testing tools like Burp Suite for a comprehensive web assessment.
Malware Scan
Incident Response
Detects malware indicators, rogue processes, and known botnet C2 connections on Windows endpoints. Best for incident response and endpoint compromise verification workflows.
Compliance Audits
Pro Only
CIS Benchmarks, DISA STIGs, PCI DSS, and HIPAA templates produce control-by-control pass/fail compliance results. Navigate: Scans → New Scan → Compliance. Requires Nessus Professional.
Creating a Custom Scan Policy
01
Open Policies
Navigate to Scans → Policies → New Policy → Advanced Scan. Give the policy a descriptive name for the assessment type.
02
Configure Discovery and Assessment
Under Discovery: set port scan range (1-65535 for thorough). Under Assessment: enable Safe Checks, set intensity. Under Performance: set max concurrent hosts and checks.
03
Select Plugin Families
Under Plugins: enable the families you need (Backdoors, CGI, Databases, Windows, Service Detection). Always disable Denial of Service on any production environment scan. Save the policy to reuse across multiple scans.

Conducting Vulnerability Scans

The quality of a Nessus scan is determined primarily by two factors: credential configuration and target accuracy. Unauthenticated scans detect only externally visible vulnerabilities -- they miss missing patches, weak configurations, and local privilege escalation paths that are only visible from inside the system. Credentialed scans can find 5-10x more vulnerabilities on the same target because they access the OS, registry, installed software list, and configuration files directly. Always run credentialed scans when the engagement scope permits.

01
Create a New Scan
Navigate to Scans → New Scan and choose your template. In Basic Settings: give the scan a descriptive name (e.g. "Internal VA Oct -- 10.0.0.0/24"). Enter targets as IP addresses, CIDR ranges, or hostnames -- one per line or comma-separated. Configure schedule (once, daily, or weekly) and email notifications for scan completion alerts.
02
Configure Windows Credentials (SMB)
Click the Credentials tab in the scan. Click Add → Windows. Enter username (DOMAINdministrator or .\localadmin) and password. Target requirements: SMB port 445 open, File and Printer Sharing enabled, Remote Registry running, firewall exception for SMB.
Credentialed scans find 5-10x more vulnerabilities than unauthenticated scans on the same target.
03
Configure Linux/Unix Credentials (SSH)
Click Add → SSH. Enter username (root or a sudo-capable account) and choose authentication method: Password or SSH private key. For sudo users, set Elevate privileges with: sudo and enter the sudo binary path. The scan account should have NOPASSWD configured in sudoers for uninterrupted scanning.
04
Performance and Safety Settings
Under Settings → Assessment: ensure Safe Checks is ON and DoS plugins are DISABLED for any production target. Under Settings → Performance: Max Concurrent Checks -- 5 for production, 10 for dedicated lab. Max Concurrent Hosts -- 5 for production, 30 for a dedicated scan network. Schedule during maintenance windows.
Always enable Safe Checks on production. Never enable Denial of Service plugins on live business systems.
[ ! ]

Production Scanning Safety. Always disable DoS plugins before scanning production systems. Set Max Concurrent Checks to 3-5 for production environments. Schedule scans during maintenance windows when possible. Ensure Safe Checks is enabled. Confirm written authorisation covers the target IPs before starting any scan -- unanticipated scanning even within a network can violate engagement scope.

Analysing Scan Results

Nessus scan results are structured around five severity levels based on the Common Vulnerability Scoring System (CVSS). Understanding how to triage and prioritise these findings is critical -- a large organisation may have thousands of findings across hundreds of systems, and effective prioritisation separates actionable security intelligence from overwhelming noise. Start with Critical findings on externally accessible systems; these represent the highest business risk and the most likely active exploitation paths.

Critical9.0 - 10.0Remote code execution, authentication bypass, publicly exploitable CVEs
High7.0 - 8.9Privilege escalation, sensitive data exposure, significant service impact
Medium4.0 - 6.9Requires user interaction, limited impact scope, information disclosure
Low0.1 - 3.9Minimal impact, hardening improvements, defence-in-depth controls
Info0.0Version banners, port listings, best practice notes -- no direct risk
📋 View by Host
The default view after scan completion. Left panel shows each scanned host with colour-coded bars representing Critical/High/Medium/Low/Info counts. Click any host to see all vulnerabilities discovered on that specific system. Best for host-by-host remediation planning.
📊 View by Vulnerability
Click the Vulnerabilities tab in scan results. All findings sorted by severity -- Critical first. Each row shows Plugin ID, severity badge, vulnerability name, and number of affected hosts. Best for identifying the highest-impact issues across the entire scan scope.
🔍 Filter Bar
The filter bar at the top of results lets you filter by: Severity (Critical/High/Medium/Low/Info), VPR Score (Tenable's threat-intel priority rating), Plugin Family (Windows/CGI/Databases), Port, or Plugin ID for targeted review.
📄 Reading a Finding
Click any vulnerability to expand its full detail panel. You will see: Description (what the vulnerability is), Solution (how to fix it), CVSS Score and vector, CVE References (linked to NVD), Plugin Output (the actual scan evidence), and See Also (vendor advisories).
✅ Verifying Findings
Always verify using the Plugin Output field -- this is the actual evidence captured during scanning. A banner grab showing "Apache/2.4.49" confirms the vulnerable version. A registry value confirms an unpatched Windows KB. Evidence in the plugin output eliminates false positives before reporting.
💾 Export Formats
Click Export (top right of results). Choose: PDF (client-ready report), HTML (interactive browser report), CSV (spreadsheet analysis and prioritisation), Nessus XML (import/API processing), or JSON (programmatic AI pipeline integration).

Professional Reporting

Nessus generates professional vulnerability assessment reports directly from scan data. The quality of a Nessus report -- and the remediation action it drives -- depends on proper configuration of report scope, audience-appropriate filtering, and intelligent presentation of findings. A raw Nessus export containing thousands of Info-level findings alongside Critical vulnerabilities is not a useful deliverable; a filtered, prioritised report presenting only actionable findings with clear remediation guidance is.

01
Generate an Executive Summary Report
Navigate to Scans → [completed scan] → Export. Choose PDF format. In Report Configuration set Min Severity to High. Group by Severity. Enable Charts and Statistics and Host counts. Disable individual plugin output. This produces a business-risk-focused document for C-suite and management audiences -- no technical jargon, clear risk context.
02
Generate a Technical Remediation Report
Choose PDF or HTML format. Set Min Severity to Medium (Critical + High + Medium only). Group by Host (one section per system). Enable full plugin output, CVE references, and solution steps. This gives the IT/security operations team everything they need to understand, prioritise, and fix each finding without needing to access Nessus directly.
03
Export a CSV for Remediation Tracking
Choose CSV format. This exports columns: Host, Plugin ID, Severity, CVE, Solution, and CVSS Score. Open in Excel or Google Sheets. Sort by Severity DESC then CVSS DESC. Use this as a prioritised remediation task list that patch management teams can work through and update as items are resolved.
CSV is the most flexible format -- import directly into ServiceNow, Jira, or any ticketing system for remediation workflow management.
04
Compliance Audit Reports (Pro)
After running a CIS, DISA STIG, or PCI DSS compliance scan, export the Compliance chapter. Each control appears with a PASSED / FAILED / WARNING / ERROR status, the control reference number (CIS 1.1.1 or STIG V-12345), and the actual system value that caused the failure. Ready-to-submit audit evidence for regulatory assessments.
05
Compare Scans for Remediation Progress
In Scans → [scan] → Compare, select two scan instances (e.g. last month vs this month). Nessus shows Remediated (fixed since last scan), New (appeared since last scan), and Unchanged findings. This is your evidence of remediation progress for management reporting and SLA tracking.

Advanced Scanning Techniques

Advanced Nessus techniques separate professional vulnerability assessors from basic users. Agent-based scanning deploys lightweight Nessus agents on endpoints that scan locally and report to the manager -- ideal for remote workers and DMZ systems that cannot be reached by network-based scanning. Compliance auditing tests systems against configuration benchmarks (CIS, DISA STIG, PCI DSS) and produces control-by-control pass/fail results. Plugin customisation allows writing targeted checks for organisation-specific vulnerabilities.

💻 Nessus Agent Deployment
Download the Nessus Agent from tenable.com/downloads. Install on the endpoint (Linux: .deb/.rpm, Windows: .msi). After install, link the agent to your Nessus Manager via the agent settings -- enter the Linking Key (found in Nessus under Settings → Sensors → Nessus Agents) and the manager hostname. The agent then appears in Nessus and can be included in agent scans without network connectivity to the target.
✅ Compliance Audit Setup
Navigate to Scans → New Scan → Compliance. Choose your standard: CIS Level 1 or 2, DISA STIG (OS-specific), PCI DSS, or HIPAA. Add target credentials (SSH for Linux, WinRM for Windows). Each control reports as: PASSED, FAILED (with current value shown), WARNING (needs manual review), or ERROR (could not be tested).
🛠️ Custom Audit Files
Create a custom .audit file with organisation-specific compliance checks using Nessus audit syntax. Define checks that test command output, registry values, file content, or service states against expected values. Upload via Policies → [policy] → Compliance → Add file. Enables bespoke compliance controls beyond the built-in standards.
🌐 Web Application Scanning
Use the Web Application Tests scan template. In the scan settings, enable Follow Redirects, Test for common web flaws (XSS, SQLi, CSRF), and set thoroughness level. Provide HTTP login credentials for authenticated scanning. Combine with Burp Suite for comprehensive coverage -- Nessus for breadth, Burp for depth.

REST API & Automation

The Nessus REST API enables full programmatic control of Nessus -- launching scans, monitoring progress, downloading results, and integrating Nessus data into SIEM, SOAR, and vulnerability management platforms. The API uses JSON over HTTPS and authenticates with API keys generated in the Nessus settings. Automating Nessus via the API is essential for mature security programmes that need continuous scanning, integration with ticketing systems, and automated remediation tracking.

#!/usr/bin/env python3 # Nessus REST API automation # Generate API keys: Settings > My Account > API Keys > Generate import requests, json, time, urllib3 urllib3.disable_warnings() # suppress self-signed cert warning # ── CONFIGURATION ───────────────────────────────────── NESSUS_URL = "https://localhost:8834" ACCESS_KEY = "your-access-key" SECRET_KEY = "your-secret-key" HEADERS = {"X-ApiKeys": f"accessKey={ACCESS_KEY}; secretKey={SECRET_KEY}", "Content-Type": "application/json"} def api(method, endpoint, data=None): url = NESSUS_URL + endpoint resp = requests.request(method, url, headers=HEADERS, json=data, verify=False) resp.raise_for_status() return resp.json() if resp.text else {} # ── LIST ALL SCANS ──────────────────────────────────── scans = api("GET", "/scans") for s in scans.get("scans", []): print(f" [{s['status']:<12}] {s['id']:>6} {s['name']}") # ── LAUNCH A SCAN ──────────────────────────────────── def launch_scan(scan_id): result = api("POST", f"/scans/{scan_id}/launch") return result.get("scan_uuid") # ── WAIT FOR SCAN COMPLETION ────────────────────────── def wait_for_scan(scan_id, poll_interval=30): print(f"[*] Monitoring scan {scan_id}...") while True: detail = api("GET", f"/scans/{scan_id}") status = detail.get("info", {}).get("status", "unknown") print(f" Status: {status}") if status in ("completed", "canceled", "aborted"): break time.sleep(poll_interval) return status # ── EXPORT SCAN RESULTS AS CSV ──────────────────────── def export_scan(scan_id, fmt="csv"): export = api("POST", f"/scans/{scan_id}/export", data={"format": fmt, "chapters": "vuln_hosts_summary"}) file_id = export["file"] while True: status = api("GET", f"/scans/{scan_id}/export/{file_id}/status") if status.get("status") == "ready": break time.sleep(5) resp = requests.get(f"{NESSUS_URL}/scans/{scan_id}/export/{file_id}/download", headers=HEADERS, verify=False) fname = f"nessus_scan_{scan_id}.{fmt}" with open(fname, "wb") as f: f.write(resp.content) print(f"[+] Exported: {fname}") return fname

AI Integration in Nessus Workflows

AI transforms Nessus from a scanning tool into an intelligent vulnerability intelligence platform. Nessus produces vast amounts of structured data -- thousands of findings, CVE references, CVSS scores, and plugin output -- that are ideal inputs for LLM analysis. AI can prioritise findings based on real-world exploitability beyond CVSS scores, generate executive-ready narratives from raw vulnerability data, correlate findings across hosts to identify systemic weaknesses, and produce remediation roadmaps that patch management teams can actually execute. The combination of Nessus's comprehensive scanning with GPT reasoning produces security intelligence at a level previously requiring a team of senior analysts.

[ AI ]

Security teams using AI-augmented Nessus workflows reduce vulnerability report writing time by 70% and improve remediation prioritisation accuracy by correlating Nessus findings with real-world threat intelligence, exploit availability, and asset criticality -- producing risk scores that CVSS alone cannot capture.

📊
Finding Prioritisation
AI analyses Nessus CSV/JSON exports and re-prioritises findings beyond raw CVSS scores by incorporating exploit availability (Metasploit module exists?), asset criticality (internet-facing vs internal), and threat intelligence (actively exploited in the wild?). Produces a prioritised remediation queue where the most dangerous vulnerabilities on the most exposed systems appear first, regardless of CVSS score ordering.
📄
Report Generation
AI converts Nessus scan data into professional vulnerability assessment reports -- executive summaries with business risk narratives, technical finding sections with remediation steps, and compliance gap analysis. Transforms a raw CSV export with 2,000 rows into a structured, audience-appropriate report in minutes. Also generates management-level risk dashboards and remediation roadmaps with effort estimates per finding group.
🔍
Vulnerability Correlation
AI identifies systemic vulnerability patterns across Nessus scan results that individual finding review misses -- "14 hosts have unpatched Apache 2.4.49, 8 of which are internet-facing and share a common service account. CVE-2021-41773 is actively exploited. This represents a critical attack chain from external to internal systems through a single shared credential." Attack chain analysis that no automated scanner provides natively.
🛠️
Custom Plugin Generation
LLMs generate custom Nessus audit files and plugin NASL (Nessus Attack Scripting Language) code for organisation-specific checks -- "write a Nessus audit check that verifies our custom application's authentication header is present and meets our internal policy." Enables security teams to extend Nessus beyond its built-in 180,000 plugins for proprietary systems and internal compliance requirements.
🔴
Remediation Planning
AI generates structured remediation plans from Nessus findings -- grouped by technology, prioritised by risk, with effort estimates and dependencies identified. "Patch these 47 Windows systems in this order because they share the same vulnerability and patching them together minimises downtime." Converts a vulnerability list into an actionable project plan that IT operations teams can execute without security expertise.
📉
Trend Analysis
AI analyses multiple Nessus scans over time to identify vulnerability trends, measure remediation velocity, and predict future risk posture. "Critical vulnerability count increased 23% last month due to new cloud assets being added without security review. At current remediation rate, the backlog will grow by 40% in 90 days." Converts point-in-time scan data into strategic security programme intelligence for management reporting.

" "Analyse this Nessus scan summary and provide: " "1. RISK EXECUTIVE SUMMARY -- 3 sentences for C-suite audience " "2. CRITICAL ATTACK PATHS -- which findings chain to compromise? " "3. PRIORITISED REMEDIATION -- top 10 actions, ordered by impact " "4. SYSTEMIC WEAKNESSES -- patterns across multiple hosts " "5. QUICK WINS -- findings fixable in under 1 hour " "Nessus data: " + json.dumps(data, indent=2) ) r = openai.OpenAI().chat.completions.create( model="gpt-4o", messages=[{"role":"user","content":prompt}] ) return r.choices[0].message.content csv_path = sys.argv[1] if len(sys.argv) > 1 else "nessus_export.csv" data = parse_nessus_csv(csv_path) print(f"[*] {data['total_findings']} findings across {data['unique_hosts']} hosts") print(f"[*] Severity: {data['severity_counts']} ") print(ai_analysis(data))
